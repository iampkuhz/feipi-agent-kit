# 信息充足性检查

## 通用最低要求

一页 PPT 至少需要：

1. **主题**：这一页讲什么。
2. **一句话结论**：希望 CTO 看完记住什么。
3. **原始事实**：支撑结论的事实、数据、模块、流程、对象、对比项。
4. **关系结构**：这些内容之间是什么关系（分层、流程、对比、因果、阶段、取舍）。

四项齐全才能生成 Page Contract。任何一项缺失，视为信息不足。

## 不同页面类型的最低要求

### 架构图页（Architecture Map）

需要：

- 模块/组件清单。
- 模块之间关系（依赖、数据流、控制流）。
- 需要强调的边界或瓶颈。

**缺少模块关系时，不生成。**

### 流程图页（Flow Diagram）

需要：

- 角色。
- 步骤。
- 顺序。
- 输入输出。
- 关键判断点。

**缺少步骤顺序时，不生成。**

### 对比页（Comparison Matrix）

需要：

- 对比对象。
- 对比维度。
- 每个对象在各维度下的事实。
- 结论或倾向。

**缺少对比维度或对象事实时，不生成。**

### 路线图页（Roadmap Timeline）

需要：

- 阶段。
- 每阶段目标。
- 每阶段关键事项。
- 阶段之间递进关系。

**缺少阶段目标时，不生成。**

### 指标页（Metrics Dashboard）

需要：

- 指标名称。
- 指标值。
- 指标含义。
- 指标之间关系或结论。

**缺少指标值时，不要自造。**

### 分层能力页（Layered Stack）

需要：

- 层级。
- 每层能力。
- 层级边界。
- 依赖关系。

**缺少边界时，需要用户补充。**

### 决策页（Decision Tree）

需要：

- 决策问题。
- 可选方案。
- 判断标准。
- 取舍逻辑。
- 推荐结论或待决点。

**缺少判断标准时，不生成。**

### 问题归因页（Problem Attribution）

需要：

- 问题。
- 现象。
- 原因。
- 影响。
- 解决路径。

**缺少原因或影响时，需要补充。**

## 信息不足时输出模板

```
当前信息不足，无法稳定生成 PPT 单页。请补充以下原始信息：

1. 缺少【核心结论】：这一页到底想让 CTO 记住什么？
2. 缺少【关键对象】：需要展示/比较/连接的对象有哪些？
3. 缺少【支撑事实】：每个对象有哪些数据、特征、关系或依据？

请直接补充原始信息，不需要描述设计风格。
```

根据实际缺少项调整模板中的具体条目，但保持格式一致。
