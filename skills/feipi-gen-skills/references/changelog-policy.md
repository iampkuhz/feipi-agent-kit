# 变更记录与 README 同步

## CHANGELOG 规则
- 每次更新完成后，必须更新仓库根目录 `CHANGELOG.md`。
- 按天分章节，格式 `## YYYY-MM-DD`，时间倒序排列（新日期在上）。
- **极致精简（强制）**：
  - 同一 SKILL 在同一天只允许 **1 条**记录；必要时最多 **2 条**（功能+修复）。
  - 单条记录不超过 **12 个汉字**（含英文/数字），必须可合并就合并。
  - 允许抽象总结，不写细节、不列清单、不罗列子项。
  - 当天所有更新总条数建议 ≤ 5；超出必须再合并压缩。
- 若是继续优化昨天的内容，在昨天对应条目追加“未调试成功”标记；最多只回看一天，更早内容不做修改。

## README 同步
- 每次更新后，检查 `README.md` 是否需要微调（命令、示例、路径等变更）。
