# 变更记录与 README 同步

## CHANGELOG 规则
- 每次更新完成后，必须更新仓库根目录 `CHANGELOG.md`。
- `CHANGELOG.md` 只按日期做二级标题，格式 `## YYYY-MM-DD`，时间倒序排列（新日期在上）。
- 版本来源是目标 skill 的 `agents/openai.yaml` 顶层 `version` 字段；更新 skill 时先递增版本，再写 changelog。
- 推荐结构：

```md
# 变更记录

## YYYY-MM-DD
- <skill-name> v<version>：合并后的更新摘要
- <another-skill> v<version>：合并后的更新摘要
```

- 每条记录都要写清 `skill-name`、`v<version>` 和本次合并后的更新摘要。
- **极致精简（强制）**：
  - 同一天同一个 skill 只允许 **1 条**记录，也只允许升级 **1 次**版本。
  - 若同一天内该 skill 改动多次，必须合并到同一条记录中，不新增第二条、不再额外升版。
  - 单条记录只写 **1 行**；冒号后的摘要建议不超过 **18 个汉字**，最多不超过 **24 个汉字**。
  - 优先使用 **2-3 个结果短语**合并表达，如“补强版本规则与摘要约束”。
  - 禁止写成长句解释、因果链、实现过程或“新增规则：……并同步……”这类展开句。
  - 能合并就合并，优先写结果，不写过程。
- 若本次只改仓库级公共规则、未改任何 skill 自身，可在对应日期下单独写仓库级记录，但不要虚构 skill 版本。

## README 同步
- 每次更新后，检查 `README.md` 是否需要微调（命令、示例、路径、版本维护说明等变更）。
