# 绘图规则

## 组件与层

- 每个组件必须用 `as <components[].id>` 声明 alias。
- 层容器直接使用 `package "层名称" #颜色 { ... }`，不要给层再额外起 alias。
- 不得新增 brief 中没有定义的核心组件；如确需补辅助说明，只能用注释解释，不能加新图元。

## 流程

- 每条流程都要在箭头标签里落地 `Sx + 描述`。
- 标签过长时允许插入 `\\n` 换行，但不能删掉编号或改写关键命名。
- 默认使用从上到下布局，箭头优先保持跨层方向清晰，而不是追求过度紧凑。

## 布局

- 必须包含 `top to bottom direction`。
- 建议同时设置 `skinparam nodesep` 与 `skinparam ranksep`。
- `layout.include_legend=true` 时必须包含 `legend`。
- 行过长时优先拆标签或拆子图，不要把所有内容硬塞在一张超宽图里。

## 交付前检查

1. 组件 alias 是否与 brief 中的 `id` 一一对应。
2. 层名、组件名、流程编号、流程描述是否全部落图。
3. 图中是否出现 brief 未定义的核心组件。
4. 渲染图是否存在明显重叠、拥挤或超宽。
